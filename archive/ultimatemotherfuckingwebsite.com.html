Abbreviations and explanations

HTML
short for Hyper Text Markup Language, previously used by a load of amateurs to poorly structure their web page. Used to perfection here.
CSS
short for Cascading Style Sheets. Use to make the site look less ugly. Unfortunately it is also used by amateurs who fuck everything up and make the web less accessible.
WAI-ARIA
short for Web Accessibility Initiative - Accessible Rich Internet Applications, used to make stuff accessible for everybody. That is probably why you have never heard of it, you don't care about people with disabilities now do you?
JS
short for JavaScript. Some people try and make the web less boring using this programming language, but we didn't do any of that here as we are trying to prove a point!
Screen Reader
A screen reader converts the HTML on the page into speech or text with additional context for people who use assistive tech. The HTML, WAI-ARIA and to some extent the CSS used on a site can have a big impact on people who use assistive technology and you end up adding unnecessary barriers if you code it poorly, just like all the other crappy motherfucking websites before this one.
REM units
R stands for Root (as in the document root). "EM" doesn't stand for anything and nobody seems to know what it actually means in a way you could possibly use, but 1rem is 16px by default and that is enough information for now surely? Here, go read about REM units here.
WCAG
short for Web Content Accessibility Guidelines. Basically a set of guidelines on how to make your website accessible. The "AA" part just refers to the level of compliance, "AAA" being the best. WCAG is a bit wordy but all the documentation is there, so what is the fucking excuse? Go read some of it now!
